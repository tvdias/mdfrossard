## 🖼️ Image Optimization & Assets (CRITICAL)

### 1. High Performance Images
Always use the `<picture>` tag to provide **WebP** as the primary format and **JPG** as a fallback. This is non-negotiable for landing pages and high-traffic sections.

```html
<picture>
    <source srcset="img/hero.webp" type="image/webp">
    <img src="img/hero.jpg" alt="Description" loading="lazy" decoding="async">
</picture>
```

### 2. Local Asset Policy
**NEVER** leave external CDN links for production images. 
- When creating content, identify external images, download them, and save them locally in the repository.
- Recommended path for landing page images: `source/l/<page-name>/img/`
- Always verify the file headers. If downloading WebP from a CDN, ensure it is a real WebP (RIFF...WEBP header) and not just a renamed JPG.

## 📄 Landing Pages (Section `/l/`)

Landing pages are independent, high-conversion pages located in `source/l/`.

1. **Location:** `source/l/<slug>/index.html`
2. **Template Base:** Always use `source/l/exemplo/` as your code base reference.
3. **Layout:** Must specify `layout: landing_page` in the frontmatter.
4. **Typography:** Use the `google_fonts` frontmatter variable instead of inline `<link>`. The layout will handle preconnect and optimized loading.
5. **Tracking:** The `landing_page` layout includes Google Tag and FB Pixel automatically.
6. **SEO:** Ensure a unique `title` and `description` are provided.

### Example Frontmatter:
```yaml
---
layout: landing_page
title: "Dentista na Barra da Tijuca | MD Frossard"
description: "Atendimento odontológico de elite com 38 anos de tradição."
google_fonts: "Adamina&family=Roboto:wght@300;400;500;700"
---
```

## 🛠️ Common Workflows

### Creating a Landing Page
1. Create folder `source/l/new-page/` and `source/l/new-page/img/`.
2. Create `index.html` using `layout: landing_page`.
3. Download all necessary images to the `img/` folder.
4. Implement content using `<picture>` for all images.
5. Verify with `npm start` and check http://localhost:4000/l/new-page/.

### Updating Blog Posts
- Files are in `source/_posts/*.md`.
- Ensure images are in `source/images/` or the specific post folder.
- Always use Brazilian Portuguese (`pt_BR`).

## 🚨 Constraints & Best Practices

1. **Language:** All user-facing content must be in **Brazilian Portuguese (pt_BR)**.
2. **Passthrough Copy:** If adding a new folder for assets, ensure it is registered in `.eleventy.js` under `contentAssetGlobs` or `addPassthroughCopy`.
3. **No Placeholders:** Never use placeholder images. Download real clinical or relevant assets.
4. **Git:** Always commit with clear, descriptive messages in English.
