## Building and Running the Site

### Prerequisites
- Node.js and npm installed
- Git for version control

### Verification Process (MANDATORY AFTER EVERY CHANGE)

1. **Build Verification:**
   ```bash
   npm run build
   ```
   - ✅ Must complete successfully with no errors
   - Check that `public/` directory is generated
   - Verify build output shows no critical errors

2. **Runtime Verification:**
   ```bash
   npm start:app
   ```
   - ✅ Server must start on http://localhost:4000
   - No errors in terminal output

3. **Browser Testing:**
   - Open http://localhost:4000
   - Verify homepage loads correctly
   - Check recently modified pages render correctly
   - Open browser console (F12) and verify no errors
   - Test links if you modified them

4. **Types of Verification to Perform:**
   - Markdown files properly formatted (if edited posts)
   - Links working (both internal and external)
   - YAML syntax valid (if modified _config.yml)
   - CSS/JS properly formatted (if modified theme files)

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Build fails | Run `npx hexo clean` then `npm run build` |
| Port 4000 in use | Kill the process using port 4000 or wait |
| YAML syntax error | Check indentation in `_config.yml` - must be spaces, not tabs |
| Markdown not rendering | Validate front matter YAML at top of post |
| Links broken | Verify permalink structure in `_config.yml` |
| Theme changes not visible | Try `npx hexo clean` and rebuild |
| Functions not working | Ensure `npm start:functions` is running (port 9000) |

---

## If You Encounter Issues

1. Check the error message carefully - it usually indicates the problem
2. Verify file syntax (YAML for configs, Markdown for posts)
3. Ensure dependencies are installed: `npm install`
4. Try clearing cache: `npx hexo clean` 
5. Verify ports 4000 and 9000 are available
6. Check git status: `git status`
7. Review recent changes: `git log --oneline -10`
